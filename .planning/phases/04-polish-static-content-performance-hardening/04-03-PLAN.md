<objective>
Ship static OG images for `/` and `/bips` per D-17. The dynamic `opengraph-image.tsx` pattern stays scoped to `/bip/[slug]` only — static pages get static PNGs (no runtime Inter font bundling, no edge-rendering cost, designer-controlled).

Purpose: When users share homepage or /bips links on Twitter / LinkedIn / Slack / Discord, the link previews render a branded card matching the BipHub visual identity (dark-navy + gold + 11-star LogoMark + page-specific title).

- `public/og-home.png` (1200×630, "BipHub — The Erasmus+ BIP directory")
- `public/og-bips.png` (1200×630, "Browse all BIPs · BipHub")
- `scripts/og-template.html` — HTML template a contributor opens in a browser, screenshots at 1200×630, exports as PNG
- `metadata.openGraph.images` added to both pages

INFO-02 (FAQ on `/what-is-a-bip`) is covered entirely by Plan 04-01; this plan does not modify `/what-is-a-bip`. The `requirements:` frontmatter is therefore empty by design.
