# Phase 1 ‚ÄĒ UI Design Contract

> Visual and interaction contract for the student-facing discovery surface of BipHub. Generated by gsd-ui-researcher; verified by gsd-ui-checker.
>
> Phase 1 visual surfaces in scope: root layout, `(public)` route group layout (StickyNav + Footer), homepage sections (Hero, EuropeMap, CategoriesBar, StatsSection, RecentBips, HowItWorks, UniversityCTA), `/bips` browse, `/bip/[slug]` detail, per-BIP `opengraph-image.tsx`. **Out of scope:** auth pages, coordinator dashboard, submission wizard, admin panel, GDPR consent banner, `/what-is-a-bip`.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom Tailwind v4 token layer + shadcn/ui CLI v4 components installed √†-la-carte) |
| Preset | not applicable ‚ÄĒ Tailwind v4 is CSS-first (`@theme` block in `app/globals.css`); no `tailwind.config.js` |
| Component library | shadcn/ui CLI v4 (Radix primitives) ‚ÄĒ only Dialog, Popover/Drawer, Toast/Sonner, Select, Skeleton, Tooltip used in Phase 1 |
| Icon library | Tabler Icons via `@tabler/icons-react` (matches mockup, tree-shakeable, MIT) |
| Font | Inter (400/500/600/700) self-hosted via `next/font/google` ‚ÄĒ never `<link>` to fonts.googleapis.com (GDPR / Pitfall 9) |

**Tailwind v4 safelist (mandatory ‚ÄĒ Pitfall 13 / D-07):**
The six choropleth fill classes must be statically discoverable. Either embed all six full class strings inside `lib/map/bins.ts` lookup objects (preferred), OR add an explicit `@source inline()` directive in `app/globals.css`. Never concatenate with template literals.

```
fill-bip-tier-0  fill-bip-tier-1  fill-bip-tier-2
fill-bip-tier-3  fill-bip-tier-4  fill-bip-tier-5
```

**Animation discipline (Pitfall 12):**
Every `motion` import comes from `motion/react`. Every animated subtree is wrapped in `<LazyMotion features={domAnimation} strict>` and uses `m.div` (not `motion.div`). Floor: ~4.6KB initial vs 34KB. Server Components must NEVER import from `motion/react`.

## Spacing Scale

The mockup uses a hybrid scale that is mostly 4-multiple-aligned with a few CSS-padding-style values that round up. Codify a strict 4-point scale and round mockup values to nearest token at implementation time.

| Token | Value | Usage |
|-------|-------|-------|
| xs   | 4px  | Icon gaps inside chips/tags, `bip-card-meta` icon-to-label gap |
| sm   | 8px  | Compact spacing, gap between chip groups, eyebrow leading-dash gap |
| md   | 16px | Default body spacing, container side padding (mobile), card content margins |
| lg   | 24px | Container side padding (desktop), hero CTA cluster gap, stat-card padding-x |
| xl   | 32px | Map wrapper padding, recent-grid card gap (rounded from mockup's 20px), card-grid gaps |
| 2xl  | 48px | Footer column gap, uni-cta two-column gap, recent-section header gap |
| 3xl  | 64px | Mobile section padding (`<600px` breakpoint), nav-to-hero internal spacing |
| 4xl  | 96px | **Desktop vertical section padding (locked by CLAUDE.md)** |

**Exceptions (all approved ‚ÄĒ derived from mockup):**

- **Nav height:** 68px (matches mockup; not in 4-point scale but is the locked sticky-nav dimension).
- **BIP card gradient header height:** 140px.
- **Stat number font size:** 44px (typography scale, see below).
- **Hero top padding desktop:** 100px (mockup `padding: 100px 0 80px` ‚ÄĒ round to 96/80 in Tailwind: `pt-24 pb-20`).
- **Touch targets:** Heart bookmark, share button, deadline-pill click region must be ‚Č•44√ó44px on mobile (WCAG / FOUN-03). Visual icon may be smaller; pad the click region.
- **Detail page sticky sidebar width:** 340px (D-09).
- **`/bips` filter sidebar width:** 280px (D-01).

**Container max-width:** `1200px` (mockup `--container`). Side padding `24px` desktop, `16px` mobile.

**Phase 1 has no destructive student-facing actions.** Bookmark removal is silent (toast confirms; no modal). Filter "Clear all" is silent. The only "warning" copy in the entire phase is the deadline-passed badge: `Applications closed` ‚ÄĒ not destructive, informational.

## Interaction Contracts

### Animations & motion

All animations use `motion` (v12, from `motion/react`) wrapped in `<LazyMotion features={domAnimation} strict>` per Pitfall 12. Pure-CSS animations (hover transforms on cards, scroll-bounce on hero indicator) do not need `motion`.

| Animation | Duration | Easing | Trigger | Reduced-motion behavior |
|-----------|----------|--------|---------|------------------------|
| Card hover lift | 200ms | ease | hover | Skipped (no transform) |
| Button hover translateY | 200ms | ease | hover | Skipped |
| Map country fill change | 200ms | ease | hover | Instant |
| Stat count-up | ~1200ms | linear | `useInView` ‚Č•0.5 threshold | Render final number immediately |
| Tooltip fade | 150ms | ease | hover | Instant fade |
| Drawer open (mobile filters) | 250ms | ease-out | tap filter button | 0ms (instant) |
| Hero scroll-bounce indicator | 2s loop | ease | always | Static (no animation) |
| Toast in/out | 200ms / 200ms | ease-out / ease-in | bookmark/share action | Instant |

### Hover states (locked)

- **Buttons primary blue:** background ‚Üí `--eu-blue-dark`, `translateY(-1px)`, `box-shadow: 0 6px 20px rgba(0,51,153,0.25)`.
- **Buttons gold:** background ‚Üí `--eu-gold-dark`, `translateY(-1px)`.
- **Buttons ghost:** border ‚Üí `--ink`, background ‚Üí `--bg-soft`.
- **Card (BipCard, cat-item):** border ‚Üí `--eu-blue`, `translateY(-2px to -3px)`, `shadow-md`.
- **Map country:** fill ‚Üí `--eu-gold !important`, stroke ‚Üí `--ink` 1px.
- **Footer link:** color ‚Üí `--eu-gold`.

### Focus states (WCAG AA ‚ÄĒ FOUN-03)

Every interactive element must have a visible focus ring distinct from hover:
- Buttons / links / inputs: `focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-eu-blue focus-visible:ring-offset-2 focus-visible:ring-offset-white` (2px ring, 2px offset).
- On dark surfaces (StatsSection, UniversityCTA, Footer): `focus-visible:ring-eu-gold` instead.
- Map country `<path>` elements: `focus-visible:outline-2 focus-visible:outline-eu-gold` AND `tabindex="0"`. The keyboard `<select>` fallback (D-08) is the primary keyboard interaction; map `<path>` focus is a secondary affordance.

### Loading

`/bips`: skeleton card grid (12 ghost cards).
Map: `<MapSkeleton>` while dynamic chunk loads.
StatsSection: numbers render `0` until `useInView` triggers.
Bookmark island: outline state until Zustand hydrates, then snaps to localStorage state (no flicker ‚ÄĒ RSC always renders outline; client hydrates).

### Error

Map fetch failure ‚Üí keyboard `<select>` is the failure UI.
Server error on `/bips` ‚Üí Next.js `error.tsx` with copy: `Something went wrong loading BIPs. Try refreshing.` + reset button.
Server error on `/bip/[slug]` ‚Üí `notFound()` ‚Üí `<BipNotFound>`.

### Toast

shadcn Sonner. Position: bottom-center on mobile, bottom-right on desktop. Max 3 visible. Duration: 3s for confirms, 5s for warnings. No errors in Phase 1 (no destructive student actions).

## Borders, Radii, Shadows, Z-Index

**Borders:** 1px standard (`--border`); hover 1px (`--border-strong` or accent). Never 2px+ except map country stroke on selected state.

**Radii (locked):**
- 6px (`--radius-sm`): inputs, small chips, secondary tags
- 10px (`--radius-md` ‚ÄĒ DEFAULT): icon squares, generic cards, drawer
- 16px (`--radius-lg`): BipCard, map wrap, stat cards, university CTA card
- 999px (`--radius-pill`): all buttons, country/deadline/field pills, eyebrow swatches

**Shadows:**
- `--shadow-sm`: hero pill, default-rest cards (subtle)
- `--shadow-md`: card hover state, sticky bottom apply bar, popover
- `--shadow-lg`: map tooltip, drawer when open
- Primary button hover: `0 6px 20px rgba(0,51,153,0.25)` (custom, not in scale)

**Z-index ladder:**
- `0`: page content
- `5`: map tooltip (within map container)
- `10`: BipCard hover overlay (none in v1, reserved)
- `30`: mobile sticky bottom apply bar
- `40`: filter drawer scrim
- `50`: sticky nav (locked by mockup)
- `60`: drawer panel
- `70`: shadcn Dialog
- `100`: toast
