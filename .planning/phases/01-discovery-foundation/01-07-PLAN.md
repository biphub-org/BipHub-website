## Phase Goal

**As a** student who has clicked into a BIP from `/bips` (or arrived from a shared link), **I want to** read the full programme details -- description, learning outcomes, virtual component, dates, ECTS, language level, partner universities, how to apply -- and immediately share it with a friend or bookmark it for later, **so that** I can decide whether to apply through my home university's Erasmus office.

> Plan 01-07 ships the complete `/bip/[slug]` detail page satisfying DETL-01 through DETL-10 plus INFO-03 inheritance. The 2-column desktop layout with a 340px sticky sidebar (D-09) and the single-column mobile layout with a sticky-bottom Apply bar (D-10) are NEW for v1 -- they do not exist in `biphub-homepage.html`. The Web Share API + clipboard fallback (D-11) and the heart bookmark (D-12, D-13) reuse Plan 01-05's Zustand store and BookmarkHeartIsland. The per-BIP OG image at `opengraph-image.tsx` uses Satori with bundled Inter (PITFALLS Pitfall 15 -- never fetch fonts from Google in the OG runtime).

**A. `lib/utils/deadline.ts`** -- pure helper. RSC + client both use it.

State branches per UI-SPEC line 358:
- past deadline -> `{ state: 'closed', daysLeft: <0|null, label: 'Applications closed' }`
- 0 < days <= 14 -> `{ state: 'urgent', daysLeft, label: '. {n}d left' }` (clock emoji)
- days > 14 -> `{ state: 'info', daysLeft, label: '. {n}d left' }`
- null deadline -> `{ state: 'rolling', daysLeft: null, label: 'Rolling deadline' }`

Implementation: normalize both `deadline` and `now` to UTC midnight via `Date.UTC(getUTCFullYear, getUTCMonth, getUTCDate)` and divide by 86_400_000 to compute integer day delta. NO `new Date().getTime()` differences (introduces TZ jitter near midnight).

**B. `lib/queries/bipDetail.ts`** -- two exports:

1. `getBipBySlug(slug)` -- single PostgREST query embedding `host_university:universities!host_university_id(...)` AND `partners:bip_partner_universities(partner_name_raw, partner_country_raw, partner_erasmus_code_raw, university:universities(...))`. Both functions call `await createClient()` from `@/lib/supabase/server`. RLS handles authorization automatically -- anon callers only see approved rows.

**C. `lib/utils/share.ts`** -- runtime feature detection per D-11.

```typescript
/**
 * Web Share API with silent clipboard fallback (D-11 / UI-SPEC line 285).
 *
 * Behavior:
 *   - If navigator.share AND navigator.canShare?.({ url, title }) -> use share
 *   - Else if navigator.clipboard.writeText -> use clipboard
 *   - Else -> { shared: false, fallback: 'unsupported' }
 *
 * Returns shape: { shared: boolean, fallback?: 'clipboard' | 'unsupported' }.
 * Caller (ShareButton) renders the appropriate Sonner toast.
 */
export async function shareBip({
  title,
  url,
}: {
  title: string
  url: string
}): Promise<{ shared: boolean; fallback?: 'clipboard' | 'unsupported' }> {
  const data = { title, url }

  if (typeof navigator !== 'undefined' && 'share' in navigator) {
    const canShare =
      'canShare' in navigator && typeof navigator.canShare === 'function'
        ? navigator.canShare(data)
        : true
    if (canShare) {
      try {
        await navigator.share(data)
        return { shared: true }
      } catch {
        // User cancelled OR share failed -- fall through to clipboard
      }
    }
  }

  if (
    typeof navigator !== 'undefined' &&
    'clipboard' in navigator &&
    navigator.clipboard?.writeText
  ) {
    try {
      await navigator.clipboard.writeText(url)
      return { shared: true, fallback: 'clipboard' }
    } catch {
      // Clipboard write failed -- caller shows error toast
    }
  }

  return { shared: false, fallback: 'unsupported' }
}
```

```typescript
/**
 * Downloads Inter Bold + SemiBold TTF binaries from rsms/inter (OFL 1.1) into
 * public/fonts/. These are read by app/(public)/bip/[slug]/opengraph-image.tsx
 * at request time and passed to Satori's `fonts` option.
 *
 * Why bundled and not fetched at runtime: PITFALLS Pitfall 15 -- Vercel's
 * serverless OG runtime does not expose system fonts and cannot fetch from
 * fonts.googleapis.com (cross-origin / GDPR concerns + adds 200-500ms latency
 * to every OG image request).
 *
 * Run once: npm run fonts:fetch
 */
import { writeFileSync, mkdirSync } from 'node:fs'
import { dirname } from 'node:path'

const FONTS = [
  {
    name: 'inter-semibold.ttf',
    url: 'https://github.com/rsms/inter/raw/master/docs/font-files/Inter-SemiBold.ttf',
  },
  {
    name: 'inter-bold.ttf',
    url: 'https://github.com/rsms/inter/raw/master/docs/font-files/Inter-Bold.ttf',
  },
] as const

async function main() {
  const dir = 'public/fonts'
  mkdirSync(dir, { recursive: true })
  for (const f of FONTS) {
    const res = await fetch(f.url)
    if (!res.ok) throw new Error(`Failed to fetch ${f.name}: ${res.status}`)
    const buf = Buffer.from(await res.arrayBuffer())
    writeFileSync(`${dir}/${f.name}`, buf)
    // eslint-disable-next-line no-console
    console.log(`Wrote ${dir}/${f.name} (${buf.length} bytes)`)
  }
}

void main()
```
